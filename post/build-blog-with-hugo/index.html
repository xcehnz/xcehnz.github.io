摘要：本文将介绍什么是Hugo，怎么使用Hugo搭建静态网站以及如何发布到Github生成Github Pages。

什么是Hugo

Hugo是用Go语言编写的静态网站生成器。类似的静态网站生成器有Hugo、Hexo、Jekyll等。选择Hugo的原因主要是它使用起来非常简单，仅仅需要安装一个编译好的二进制文件，不需要安装一大堆依赖，而且Hugo构建静态网站的速度非常快。

GitHub Pages

使用静态网站生成器创建好静态页面后，需要把文件放到服务器上，传统的做法是购买vps虚拟主机或服务器，比如：linode、搬瓦工等。然后将生成好的静态页面上传到服务器上。当然还有更好的方式是直接将网站托管到Github Pages，可以直接通过https://username.github.io访问，稍后会进行详细讲解。

怎么用Hugo搭建博客

1 安装Hugo

Hugo目前版本是v0.59.1，可以看到有各个系统的二进制文件。

Windows

win64对应的是hugo_extended_0.59.1_Windows-64bit.zip，下载后解压到任意目录，比如D:\Hugo，为了方便在命令行里使用，将hugo.exe加到环境变量PATH目录里。

当然了如果你的windows使用Chocolatey包管理器的话，终端里输入

1

choco install hugo -confirm

MacOS

Mac里安装Hugo就很简单了，使用Homebrew，在终端里输入

1

brew instll hugo

Linux

如果你使用的是ArchLinux或Manjaro(Manjaro, Yes！👀)的话，打开终端输入

1

sudo pacman -S hugo

如果是其他发行版的Linux，使用相应的包管理器安装，或直接下载对应的版本，解压后把 hugo 文件放入 /usr/local/bin 目录并赋予权限即可。

安装完后打开终端，输入hugo verison， 如果安装成功会看到 Hugo Static Site Generator v0.59.1...的字样。

2 创建一个站点

首先创建网站目录/path/to/site，终端里输入

1
2
3

# 进入网站目录
cd /path/to/site
hugo new site test

可以看到目录里多了一个test文件夹，文件夹构成：

1
2
3
4
5
6
7
8

.
├── archetypes
├── config.toml
├── content
├── data
├── layouts
├── static
└── themes

config.toml是网站的配置属性文件，content文件夹里放你网站的内容，static目录存放静态资源如图片、js、css等。

接下来创建一篇博客文章：

1
2

cd test
hugo new post/blog-title.md

Hugo会在content目录下创建post目录，在post目录下创建blog-title.md文件，之后打开md文件，看到：

1
2
3
4
5

---
title: "Blog Title"
date: 2019-11-09T18:55:45+08:00
draft: true
---

—包裹起来的内容是yaml配置信息，叫做扉页，其中draft是true的时候，代表此文件为草稿，hugo不会真正发布这篇文章，将draft修改为false，添加博客内容，一篇博客就完成了。

3 修改主题

可以到Hugo官方的主题网站，下载自己喜欢的主题，这里用的是Even ，下载主题：

1

git clone https://github.com/olOwOlo/hugo-theme-even themes/even

在主题的 exampleSite 目录下有一个 config.toml 文件，将这个 config.toml 文件复制到你的站点目录下，根据自己的需求更改即可。

本地测试

命令行启动：

1

hugo server -t even

启动后访问http://localhost:1313，就可以看到博客内容了。

托管到Github

把网站托管到Github，需要在Github上创建一个仓库名字为username.github.io

注意：修改config.toml中的baseURL设置为https://username.github.io

在本地测试文章没有问题后，就可以发布了。终端里输入：

1

hugo -t even

Hugo会编译生成静态页面，输出到public目录。

接下来进入public目录创建为初始化git仓库，并将public目录push到Github的username.github.io目录就行了。(这里就是创建初始化git仓库并提交到github，不再做过多的赘述)

打开https://username.github.io，就可以看到你的个人网站了。
